JavaScript and the DOM

What the heck is the DOM?

The DOM (or Document Object Model) can be described as a hierarchical structure that the browser uses to make sense of everything going on in the webpage. The Document is the top most Object in the DOM. The Model is just a way of saying its lots of things all relating to one another in a tree-like structure.

Each element in the DOM is called a node, and other languages like CSS and Javascript are used to change what we see on a webpage by accessing the nodes via the DOM.

Woah, woah, woah! CSS, HTML, Javascript, I'm confused...

The difference between CSS and HTML (and JavaScript)

The three main languages used in webpage design are HTML, CSS and JavaScript. In basic terms, HTML is the structure of the page, CSS is how the page looks and JavaScript is how we interact with the page. This might make more sense if we think of them as animals. HTML is the hard working beaver who builds their dam super dooper technically sound and whilst pretty ugly (i mean it's just wood and junk) it does the job of being a damn good dam. CSS is like a unicorn. Everywhere it goes it adds glitter and sparkles and rainbows! This dam is ugly no more! JavaScript is a bit trickier to describe as an animal, but more like the little fairy that comes in and waves her little wand to create a door to get into the dam, or some fairy-lights, or a light up dance floor.

Ok, now we know enough about how HTML, CSS and JavaScript all work together through the DOM, it's time to learn some more about JavaScript and a few of the ins and outs of this marvellous language.

Control Flow

Control flow in JS (What we'll be referring to JavaScript as from now on because I'm a bit lazy) is exactly what is sounds like: the flow of the document. JS usually flows from top of the page to bottom of the page. It starts executing code at the top and finishes once it gets to the bottom. If you'll allow me to introduce another analogy: driving to the gym. Control flow of driving to the gym usually executes in the same order. We get into the car, buckle up, plug phone in to play some pump up jams, push the start button, check the mirrors, drive off.

I have said usually above since sometimes things can run against the flow. One of these instances is when we have Loops in our code.

Loops! and Loops! and Loops! and Loops!

Say you're on your way to the gym and "Dang it! I forgot my shoes!"" Better head back home and grab them. And then "Oh no! I forgot my towel" turning back around... Oh wow, we just experienced a loop! While this example of a loop is pretty annoying, loops in JS are pretty much the same! But they're not annoying - they're actually pretty useful! Loops are statements in the code that keep running on loop until there's nothing left to loop over.

let gymGear = ['bag', 'shoes', 'skipping rope']

for (i = 0; i > gymGear.length; i++) {
console.log(gymGear[i])
}

The above example shows a for loop that iterates (or moves through, one by one) through an array of things I take to the gym. The flow of the code is temporarily interrupted and the loop is executed the amount of times as there are items in the array, then regular flow is resumed. This is just one example of loops. If you're interested in reading more about loops here's a great resource.

Accessing data from Arrays vs. Objects

I briefly introduced Arrays in the previous section. An array is identified by square brackets [ ] and is a list of information, be it strings (words), numbers, or even objects or other arrays as well! Arrays store information in a list that have specific positions. (More on this soon.) Objects are like arrays in that they store data, however they are much more complex. They are used when we want to represent a "thing" in our code. For example say I wanted to represent me in my code:

let gymGear = ['bag', 'shoes', 'skipping rope']
// This is the gymGear array from before

let person = {
name: 'cat',
age: 26,
shoes: 'black',
skippingRope: 'yellow'
}
//This object called person contains information about me and my gym gear too!

As you can see, arrays and objects are similar in that they both hold information, but they are different in they uses and the information that they can hold.

One of the key things about JS is using code to get information from data stored in arrays and objects. How fun! Unfortunately you can't just use the same method to get data from both of these variables. We have to use fun things called dot-notation and bracket-notation. With Objects we can use either dot- or bracket-notation to access information in our objects. But, with arrays, we can only use bracket-notation. Here's a code example to help visualise this!

//bracket-notation for array
console.log(gymGear[0]) //logs the first (arrays begin at 0) item in our array, bag

//dot-notation for object
person.name //prints 'cat'

//bracket-notation for object
person['name'] //prints 'cat' - note the identifier has to also be a 'string'

All of these look pretty similar and they two object ones do the same thing. So why do we need two different types of notation to access objects? One reason to use bracket-notation is that our identifiers can contain spaces and begin with a number and it won't throw an error, where it would if you tried to use dot notation.

Functions

The last thing we're going to talk about today is Functions. A function in JS is a block of code that is designed to perform a particular task. They are particularly helpful when we have bits of code that we want to use more than once. Say we want to print "Hello Cat!"

function hello() {
print("Hello " + person.name + "!")
}

Here I have written a function called 'hello' that will print "Hello " and the person.name that it took from the object declared earlier, plus "!" at the end. This can be very helpful if the name of the person changed say every time a new person logged into the website. A handy tip for functions is if you are copying code you've already written then you should turn it into a function!
